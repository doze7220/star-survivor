STAR SURVIVER：エネミー・環境詳細仕様書
1. 文書の目的と設計思想
1.1 プロジェクト概要
本プロジェクト『STAR SURVIVER』は、80年代の傑作群『SubSpace』『ボスコニアン』『タイムパトロール』が持つレトロな宇宙空間の操作感と、現代の『Vampire Survivors』に代表されるローグライクな成長要素を高度に融合させた、新機軸のサバイバルシューティングである。
1.2 設計コンセプト：戦略的障害要素の定義
本作では、単なる弾幕の回避ではなく、宇宙空間の物理特性をゲームメカニクスに組み込む。
慣性物理の徹底: 全ての移動体に質量（Mass）と慣性モーメントを設定。急停止や急旋回が不可能な環境下での位置取りを重視する。
環境の武器化: 環境ギミックを障害物としてだけでなく、エネミーを殲滅するための「定置型兵器」としてプレイヤーが利用できる設計とする。
ボスコニアン・ループ: 拠点破壊が難易度スケーリングに直接影響を与えるシステムを採用し、攻勢と防勢の動的な入れ替わりを創出する。

2. 環境ギミック（マップ要素）仕様
各環境要素は物理演算に基づき、プレイヤーの重量クラス（Weight Class）に応じて異なる影響を与える。
2.1 小惑星（Asteroids）
System Logic: 耐久値（HP）を持つ動的障害物。質量に応じた3サイズ（Small/Medium/Large）を展開。
プレイヤーへの影響: 衝突時、速度の2乗に比例した衝突ダメージ。
戦略的活用法:
Requirement: 敵の射線を遮る「動く盾」として機能。
Drop Item: 破壊時、Small/MediumからはEXPジェムを、Largeからは低確率で「一時的な武装強化バフ」をドロップする。
Time Patrol要素: 特定の低高度小惑星は、ブースト移動（ジャンプ挙動）で飛び越えることが可能。
2.2 要塞（Fortresses）
System Logic: 定置型の重武装構造物。複数の独立した砲台セクションと中心核（Core）で構成。
ボスコニアン・ループ:
Requirement: 要塞の中心核を破壊すると、エリア全体の「難易度上昇タイマー」を60秒巻き戻し、一時的にエネミーの出現率を30%低下させる。
戦略的活用法: 破壊後の残骸は、巨大な遮蔽物として永続的に利用可能。
2.3 ブラックホール（Black Holes）
System Logic: 引力係数 G に基づく周囲ユニットの牽引。
Logic: 引力強度は「質量に反比例」する。小型のドローンは即座に吸い込まれるが、大型の航宙母艦は引力に抵抗して低速化するのみ。
戦略的活用法: 自機の慣性を利用したスイングバイ加速（引力圏外周での加速旋回）により、追走する敵群を振り切ると同時にイベントホライゾンへ誘導・消滅させる。
2.4 機雷原（Minefields）
System Logic: 近接信管（半径150px）を持つ爆発物。
連鎖爆発仕様: 1つの機雷が爆発すると、半径250px以内の機雷に誘爆。
戦略的活用法: 敵の大型艦を誘い込み、一箇所の機雷を狙撃することで広範囲の敵を一掃する「トラップコンボ」として機能させる。
2.5 恒星のフレア（Solar Flares）
System Logic: 30秒周期で発生する全域熱ダメージ。
シャドウ・メカニクス (Shadow Mechanic):
Requirement: フレア発生時、小惑星や要塞の「背後（光源の反対側）」に位置することでダメージを100%カットできる。
戦略的活用法: 安全地帯（Shadow Zone）を確保できない敵エネミーを焼き払う「環境清掃」として利用。
2.6 暗黒地域（Dark Regions）
System Logic: レーダー無効化および描画範囲の50%縮小。
リスクとメタ報酬:
Requirement: このエリア内でのみ、永続強化に必要な希少資源「ダークマター」がドロップする。
戦略的活用法: 視界の悪さを利用し、敵の追跡AIを攪乱（LOS：視線切り）して一時的に戦線を離脱する。

3.2 出現アルゴリズム（Difficulty Scaling）
『Vampire Survivors』型の時間蓄積式難易度に、拠点の概念を付与する。
時間スケーリング: 1分ごとにエネミーのHPが15%、弾速が5%上昇。
エリート・アップグレード:
10分経過: 防波挺が「シールド・リンク（周囲の敵を無敵化）」を獲得。
20分経過: 自爆ドローンが「分裂（爆発後に小型3機に分かれる）」を獲得。
ウェーブ・マネジメント: 特定のチェックポイント（5分、10分等）で、ボスコニアン要塞が追加出現。これを破壊しない限り、出現密度の増加ペースが加速する。

4. 環境とエネミーの相互作用（シナジー）
4.1 戦略的コンボ例
引力機雷トラップ: ブラックホールの重力圏内に機雷を散布。敵は引力で回避不能な状態になり、機雷の連鎖爆発に巻き込まれる。
ダーク・フレア・サバイバル: 暗黒地域内でフレアが発生する状況。レーダーが機能しない中、目視のみで小惑星の影（Shadow Zone）を探し当てる極限の判断を要求する。
爆光レーダーブースト: 暗黒地域内で機雷原を誘爆させた際、爆発の光によって一時的（3秒間）に画面全体の視界が200%に拡張される。
4.2 地形利用のテクニック
慣性スイング破壊: 戦艦のような重質量エネミーに対し、ブラックホールの端で急旋回。自機は加速して離脱し、追走する戦艦の慣性が引力に負けて吸い込まれるのを狙う。
小惑星プッシュ: 自機の機体（または特定の武器）で小惑星を押し、移動要塞の砲台へ「物理弾」としてぶつけることで、弾薬を消費せずに大ダメージを与える。

5. 視覚表現とレトロスタイルの維持
5.1 グラフィック指針
16色パレット制約: 『SubSpace』等のレトロ感を出すため、各スプライトの使用色を制限。ただし、弾丸やハザードは高輝度の原色を使用し、識別性を最大化する。
Pixel-Perfect Collision: 当たり判定はスプライトの見た目と厳密に一致させ、レトロゲーム特有の「ギリギリの回避」の納得感を高める。
5.2 UI/HUDへの統合
レトロなベクタースキャンおよびデジタル表示を模した警告システムを実装する。
Hazard Alert: 画面端に以下のシンボルを表示。
[ ! ] SOLAR FLARE DETECTED: 10s (恒星フレア接近)
<< GRAVITY WARNING >> (ブラックホール接近)
Radar Navigation: HUDの外周に、エネミーの質量に応じたサイズのドットを表示。
・ (小型機)
● (戦艦/母艦)
[X] (要塞)
Resource Tracker: 暗黒地域突入時、HUDのカラーが「アンバー（琥珀色）」から「グリーン」に変化し、ダークマター検知モードに切り替わったことを示す。
