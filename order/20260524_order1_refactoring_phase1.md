# 作業指示書：20260524_order1_refactoring_phase1(v0.5.03)
## 役割
あなたは優秀なフロントエンドエンジニアです。
提供したソースコードに対し、以下の「リファクタリング作業・第一段階」を行ってください。

## 目的
巨大な単一ファイル（`index.html`）を構成要素ごとに物理的に分割し、今後のリファクタリング作業の見通しと安全性を確保する。
**【絶対原則】ゲームの見た目、操作感、ロジック、変数の状態、機能は一切変更しないこと（完全互換の維持）。**

## 実装ステップと要件

### 1. CSSの分離
- 現在の `index.html` 内にある `<style>` タグの中身（CSSコード）をすべて抽出し、新規ファイル `style.css` として保存してください。
- `index.html` の `<head>` 内で、`<link rel="stylesheet" href="style.css">` を使用して読み込むように記述を置き換えてください。

### 2. JavaScriptの分離
- `index.html` 内にあるメインの `<script>` タグ（ゲームロジック本体）の中身をすべて抽出し、新規ファイル `main.js` として保存してください。
- `index.html` の `<body>` 末尾付近で、`<script src="main.js"></script>` を使用して読み込むように記述を置き換えてください。
- ※ `<head>` 内にあるエラーハンドリング用の短い `<script>` については、そのままHTMLに残すか `main.js` の冒頭に移動させるか、適切に処理してください。

### 3. ロジックの不変性の保証（厳守）
- 変数名、関数名、処理の順序、ロジックの内容は**絶対に書き換えない**でください。ES6以降の構文への修正や、コードのフォーマット変更（Prettier等の適用）もこのフェーズでは行わないでください。
- 行うのは「コードの切り出しと保存」、および「HTML側からの読み込みタグの追加・変更」のみです。

## 完了・検証条件
- `index.html`、`style.css`、`main.js` の3つのファイル構成になっていること。
- ブラウザで `index.html` を開いた際、リファクタリング前と全く同じ挙動でゲームが起動し、正常にプレイ・進行できること。
- 開発者ツールのコンソールに、ファイルの読み込みエラー等の新しいエラーが出力されていないこと。
