# 【システムプロンプト：VANGUARDRIFTER アウトゲーム＆UI 完全実装要件】

あなたは熟練のHTML5/Canvasエンジニアです。
宇宙空間シューティングゲーム『VANGUARDRIFTER - STELLAR WAR SURVIVOR -』のアルファ版における、「タイトル画面」「リザルト画面（経費精算）」「UIナビゲーションシステム」を実装してください。
画像アセットは一切使用せず、すべてHTML5 `<canvas>` とJavaScriptによるベクター描画（fillRect, arc, lineTo, fillTextなど）で表現し、各機能はモジュール化して記述してください。

## 1. 共通カラーパレットとフォント
以下のネオンカラーを使用し、暗い宇宙空間に対するハイコントラストを維持してください。フォントはすべて等幅フォント（`monospace`）を使用します。
*   **背景色:** `#000000` (完全な黒) または `rgba(0, 20, 40, 0.85)` (リザルト等での半透明ダークブルー)
*   **メイン・強調・プラス収支:** `#00FFFF` (Neon Cyan)
*   **基本テキスト・星屑:** `#FFFFFF` (Core White)
*   **マイナス収支・警告・爆発:** `#FF0044` (Enemy Red) または `#FF00FF` (Orb Magenta)

## 2. タイトル画面の実装 (`drawTitleScreen`)
以下の要素を画面中央のY軸基準で描画してください。

*   **背景アニメーション（戦場の前進感）:**
    *   **星空スクロール:** サイズと速度が異なる3階層（奥、中間、手前）の星屑（1〜2pxの白い四角）を描画し、上から下へ流し続けて前進感を演出。
    *   **遠景の爆発フレーバー:** 時々ランダムな座標で小さな爆発（中心から放射状に短い線が散り、すぐフェードアウト）を発生させる。
*   **ロゴとテキストUI:**
    *   **メインタイトル:** 中央上部に `VANGUARDRIFTER` を大きく描画し、Neon Cyanのグロー効果を付与。
    *   **サブタイトル:** その直下に `- STELLAR WAR SURVIVOR -` を描画。
    *   **アルファ版表記:** `[ ALPHA VERSION ]` と目立つ色で表記。右下に `v0.1.0 Alpha` を固定表示。
    *   **ユーザーアクション:** 画面下部に `PRESS ANY BUTTON TO SORTIE` を1Hz（0.5秒表示/非表示）で点滅表示。

## 3. リザルト画面（経費精算UI）の実装 (`drawResultScreen`)
ゲームオーバー（撃墜）時と、ミッションクリア（帰還）時に呼び出される、業務用インボイス（請求書）風の画面です。引数に `isClear`, `score`, `damageTaken`, `operationTime` (所要秒数) を取ります。

*   **精算明細（0.5秒間隔でアニメーション表示）:**
    *   **ヘッダー:** クリア時は `[ MISSION ACCOMPLISHED ]` (Neon Cyan)、撃墜時は `[ MISSION FAILED ]` (Enemy Red)。
    *   **所要時間:** `OPERATION TIME: MM:SS`。
    *   **収入 (REVENUE):** 基本報酬 (+5000 C)、撃破バウンティ (+ `score` C)、タイムボーナス (+ `operationTime * 5` C) を加算。
    *   **支出 (EXPENSES):** 機体修理費 (- `damageTaken * 10` C)。撃墜・途中撤退時はユニオン天引き（緊急回収費）として総収入の30%をマイナス。これらは Enemy Red で表示。
    *   **最終損益 (NET PROFIT):** 合計額を算出し、プラスなら Neon Cyan、マイナス（借金）なら Enemy Red で大きく表示。
*   **シエロの通信傍受演出（タイプライター効果）:**
    *   明細表示後、画面下部にオペレーターからのメッセージを1文字ずつ出力。
    *   大黒字（無傷）: `「お疲れ様ですー。今回は修理費ゼロ、完璧な仕事ですね！」`
    *   長時間生存＆無傷: `「20分オーバーの長期戦、お疲れ様です！ 無傷なんてさすがですねー！」`
    *   黒字だが修理費あり: `「あー、帰還お疲れ様です。結構機体がボロボロですねぇ……。修理費、引かせていただきますねー。」`
    *   長時間生存＆大破: `「あちゃー、欲張って残業するからですよー。はい、回収費と修理費、ドーンと引かせていただきますねー」`
    *   即撃墜（赤字）: `「あちゃー、派手にやられましたね。今月は赤字ですよ？ 死ぬ気で働いて返してくださいねー。」`
*   **アナログスタンプ演出（決済印）:**
    *   メッセージ完了後、明細の中央に斜め(-15度)の巨大スタンプをアニメーション（3倍から等倍へ0.2秒で急縮小）で押印。押された瞬間に画面全体をシェイク（振動）させる。
    *   最終損益がプラスなら `[ SETTLED ]` (Neon Cyan)。マイナスなら `[ DEBT EXECUTED ]` (太字の Enemy Red)。
*   **次への導線:**
    *   最下部に `> RETURN TO GARAGE` と `  RETRY MISSION` を表示（キーボードUI制御対象）。

## 4. キーボードUI・フォーカス管理システム
すべてのメニュー（タイトル、リザルト、レベルアップ）で、キーボード（WASD / 方向キー、Enter / Space）とマウスのハイブリッド操作を実装してください。

*   **テキストメニューの挙動（リザルト等）:**
    *   選択中の項目は左側に `[ > ]` を表示し、テキストを1Hzで明滅。
*   **カード形式メニューの挙動（レベルアップ時のパワーアップ選択）:**
    *   選択中カードは枠線を3pxに太くし、Core Whiteで高速明滅させ、1.05倍に拡大（Z-order最前面）。
    *   **動的カーソル「▼」のアニメーション:** 選択中カードの直上に下向きの三角形 `▼` (Neon Cyan) を描画。キー入力でフォーカスが切り替わった際、現在位置から次のカードの直上へ、線形補間（Lerp, t=0.2）で滑らかに横移動させる。
    *   `▼` 自身には上下にゆっくり浮遊するHoveringアニメーションを付与。

## 5. 決定アクション時のJuiciness（トランジション演出）
Enterキー等で項目を「決定」した瞬間に、以下の強烈なフィードバックを実装してください。

*   **タイトル画面での出撃決定:**
    *   テキストを即座に `>>> SORTIE INITIATED <<<` に書き換え、高速明滅（白飛び状態）させる。
    *   キャンバス全体をCore Whiteで一瞬塗りつぶし、0.5秒かけてフェードアウトするフラッシュバック演出。
    *   待機していた自機（▲）の底面から大量のパーティクルを真下に噴射し、自機を画面上部へ急加速（Ease-In）で飛び去らせる。
*   **レベルアップ画面でのカード決定:**
    *   0.5〜0.8秒間時間を止め、非選択カードを暗くして画面下部へ落下（ドロップアウト）させる。
    *   選択されたカードは画面中央へ吸い込まれ、1.5倍に拡大し枠線が激しく発光。
    *   カード上の「▼」カーソルが、カードに向かって「ガシャン！」と突き刺さるように急降下し、接触した瞬間に光のパーティクル（ドット）を四方八方へ弾けさせる。

## 6. 補足
1. レベルアップUIの描画・制御方式について
結論：Canvasベクター描画への完全移行で大賛成（GOサイン）です！
DOM要素（HTML/CSS）の限界を超え、60fpsで他のゲーム内パーティクルや描画と完全に同期した「極上のJuiciness」を実現できるなら、Canvasでゴリゴリ描画してもらうのがベストです。非選択カードがドロップアウトし、選択カードが吸い込まれて「▼」がガシャン！と突き刺さる演出は、本作の手触りを決定づける重要な要素です。ぜひCanvasで完璧に表現してもらいましょう！
2. リザルト明細（経費精算UI）の計算式とパラメータの定義
結論：すべて提案通りの仕様で実装をお願いします！
score引数の定義: 引数 score は「撃破数（GAME.killCount）」として扱い、撃破バウンティは +(score * 100) C の計算式でお願いします。
「緊急回収費」の計算基準: ご認識の通り、「基本報酬 ＋ 撃破バウンティ ＋ タイムボーナス」の**総収入合計の30%**を天引きするシビアな仕様で問題ありません。
最終損益の累積クレジットへの反映: 演出上の数値ではなく、実際に GAME.credits に加算・減算（赤字の場合は借金としてマイナス化）させてください。 企画書の仕様にも「撃墜時の修理費で資金がマイナス（借金）化。この状態では機体強化が一切不可となる。」
と定義されているため、アルファ版の段階からこの「借金の重み」をシステムとして組み込んでおきたいです。
3. シエロの通信傍受メッセージの発生しきい値
結論：提案されたしきい値で完璧です！
アルファ版のテストプレイ（短いサイクルでループを回す）を考慮し、「120秒（2分）以上」を長時間生存のトリガーとするのは非常に実用的で賢い判断です。 また、「長時間生存 ＆ 大破」の条件である「ダメージ150以上」というのも絶妙です。自機の初期HPは100に設定されている
ため、「途中で回復アイテムを拾いながらギリギリまで粘って被弾しまくった状態」を正確に拾い上げており、シエロの**「欲張って残業するからですよー」**というセリフの文脈と完璧に一致しています！
4. タイトル画面での「自機待機」と「任意のボタン」の範囲
結論：提案通りの挙動で実装をお願いします！
自機待機位置と挙動: 画面中央下部（ロゴ・テキストの下）で待機し、真下からパーティクルを噴いて上部へカッ飛んでいく（Ease-In）発進演出で間違いありません。
検知範囲: PRESS ANY BUTTON の名に恥じぬよう、Enter/Spaceに限らず「キーボードの任意のキー入力」を検知して出撃するアーケードスタイルでお願いします。
