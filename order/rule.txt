# タスク3：ハードコードの定数化 実装指示書

## 1. 目的
コード内に散在するマジックナンバー（UI座標、ゲームバランス因数、基準値）を一元管理し、調整の容易性と安全性を確保する。

## 2. 実装対象ファイル
index.html

## 3. 実装手順
1. 定数管理オブジェクト「CONFIG」の定義を拡張する。
   - UIレイアウトの基準値を格納する「UI」オブジェクトを追加。
   - ゲームバランス（スピード、クールダウン、出現頻度）を格納する「BALANCER」オブジェクトを追加。
2. ソースコード全域を検索し、以下の定数に関連する数値をCONFIGオブジェクトから参照するように書き換える。
   - UI描画時のY座標オフセット値（radarRadius関連）。
   - 通信メニューやレベルアップUIの配置基準値。
   - プレイヤー移動速度、ブースト倍率、敵出現インターバル等のバランスパラメータ。
3. 書き換え完了後、旧来の直値が残っていないか確認する。

## 4. 優先チェック項目
- 書き換え前後で、UIの表示位置にズレが発生していないか確認。
- 移動速度や出現頻度に変化がないか確認。
- 定数参照先を間違えていないか（特にY座標の計算式）。

【役割】
あなたは優秀なフロントエンドエンジニアです。
提供したソースコードに対し、敵機の移動ロジック（物理演算）の修正を行ってください。

【修正要件：敵機への慣性（ベクトル）駆動の適用】
現在、敵機の移動が座標（x, y）への直接加算で行われているため、プレイヤー機のような慣性ドリフトが発生せず、不自然な挙動になっています。敵機の駆動方法をプレイヤー機と全く同じ物理モデル（vx, vy と 摩擦係数）に修正してください。

1. **速度ベクトルの追加:**
   敵の生成時（オブジェクト初期化時）に、`vx: 0` および `vy: 0` のプロパティを追加してください。
   ※母艦からカタパルト発進する際は、この `vx` `vy` に初期の発進ベクトルを与えてください。

2. **移動ロジックのベクトル化:**
   敵の `update()` などのメインループ処理において、目標角度（`angle`）に向かって直接 `x`, `y` を操作するのではなく、以下のようにベクトルに加算するように修正してください。
   （※加速度は各敵機の性格パラメータ `speed` に依存させます）
   
   // 例（ロジックの意図として）
   const accel = this.speed * 加速度係数; // プレイヤーの移動処理とスケールを合わせる
   this.vx += Math.cos(this.angle) * accel;
   this.vy += Math.sin(this.angle) * accel;

3. **摩擦（空間抵抗）の適用:**
   プレイヤー機と同様に、毎フレーム `vx` と `vy` に摩擦係数（`FRICTION` など、プレイヤーと同等の定数）を乗算して減速させてください。
   
   this.vx *= CONFIG.FRICTION; // またはプレイヤーと同じ摩擦値
   this.vy *= CONFIG.FRICTION;

4. **座標への適用:**
   最終的に `this.x += this.vx;` および `this.y += this.vy;` で実際の座標を更新してください。
