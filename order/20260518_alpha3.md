# 『STAR SURVIVOR』コントロールシステム拡張指示書 (Alpha 3)

本ドキュメントは、自機のフライトモデル（操作性および慣性挙動）に、伝統的な宇宙慣性ドリフトシューター『SubSpace』の操作性を追加し、「マウスエイムモード」と「サブスペースモード」をキーボード操作でリアルタイムに切り替え可能にするための実装指示書である。

---

## 1. コア仕様

### 1.1 共通物理モデル
- **摩擦係数 (`CONFIG.FRICTION`)**: 双方のモードで共通の値（現在の `0.995`）を使用する。モード切り替え時に摩擦自体の係数は変更しない。

### 1.2 コントロールモードの定義と切り替え
- ゲームのステートに `GAME.controlMode = 'MOUSE_AIM';`（デフォルト）を追加する。
- プレイ中に **`C` キー（KeyC）** を押すことで、以下の2つのモードが交互に瞬時に切り替わる。
  - `'MOUSE_AIM'`（マウスエイムモード）
  - `'SUBSPACE'`（サブスペースモード）
- 切り替え成功時、画面上に短時間ポップアップUI（ポップアップテキスト）で現在のモードを表示するとさらに良い（例: `「CONTROL: MOUSE AIM」` / `「CONTROL: SUBSPACE」`）。

---

## 2. 各モードの操作仕様

### 2.1 マウスエイムモード (MOUSE_AIM) - 【既存ベース・一部拡張】
従来の360度自由エイム射撃を活かしつつ、旋回操作をキーボードにアサインしたモード。
- **旋回操作 (A / D キー)**:
  - **Aキー（左）**: 自機の機首（`player.bodyAngle`）を反時計回りに旋回させる。
  - **Dキー（右）**: 自機の機首（`player.bodyAngle`）を時計回りに旋回させる。
  - 旋回速度は一定（例: 毎フレーム `0.07` ラジアン）。
- **推力操作 (W / S キー)**:
  - **Wキー（前進）**: 現在の機首の向き（`player.bodyAngle`）に向かって推進加速度を加算。
    `player.vx += Math.cos(player.bodyAngle) * playerStats.moveSpeed;`
  - **Sキー（後退・ブレーキ）**: 現在の機首の向きの真後ろに向かって推進加速度を加算。
    `player.vx -= Math.cos(player.bodyAngle) * playerStats.moveSpeed;`
- **砲塔制御 (マウス移動)**:
  - マウスカーソルのある方向へ、砲塔（`player.turretAngle`）が360度自在に旋回・追従する。
- **射撃**:
  - 砲塔の向いている方向（`player.turretAngle`）へ弾が飛ぶ。

### 2.2 サブスペースモード (SUBSPACE) - 【新規実装】
名作『SubSpace』の挙動を再現し、機首方向へエイムと推進が完全同期する、慣性ドリフトに特化した本格宇宙機動モード。
- **旋回操作 (A / D キー)**:
  - **Aキー（左）**: 自機の機首（`player.bodyAngle`）を反時計回りに旋回させる。
  - **Dキー（右）**: 自機の機首（`player.bodyAngle`）を時計回りに旋回させる。
  - （※マウスエイムモードと同一の旋回ロジック）
- **推力操作 (W / S キー)**:
  - **Wキー（前進）**: 現在の機首の向きに向かって推進。
  - **Sキー（後退・ブレーキ）**: 現在の機首の真後ろに向かって推進。
  - （※マウスエイムモードと同一の推進ロジック）
- **マウスと機首の連動 (機体制御)**:
  - **マウスの方向へ機首（`player.bodyAngle`）が自動で追従・回転する**（マウスによる機体の旋回制御）。
  - ※ KBMのキー旋回（A/D）とマウスによる機首旋回が同時に機能し、極めてスムーズなエイム＆フライトを実現する。
- **砲塔制御 (自機正面固定)**:
  - 砲塔は完全に機体正面（`player.bodyAngle`）にロック・固定され、独立した回転は行わない。
  - `player.turretAngle = player.bodyAngle;`
- **射撃**:
  - 機首の正面方向（`player.bodyAngle`）へ真っ直ぐ弾が飛ぶ。

## 3. 実装ガイドライン（コード変更ポイント）

### 3.1 `GAME` オブジェクトへのプロパティ追加
```javascript
const GAME = {
    // ...既存のプロパティ
    controlMode: 'MOUSE_AIM', // 'MOUSE_AIM' または 'SUBSPACE'
};
```

### 3.2 入力イベントハンドラー (`keydown`)
`C` キーの押下イベントを検知し、モードをトグル切り替えする。
```javascript
window.addEventListener('keydown', e => {
    keys[e.code] = true;
    
    // Cキーによるコントロールモードの切り替え
    if (e.code === 'KeyC' && GAME.state === 'PLAYING') {
        GAME.controlMode = GAME.controlMode === 'MOUSE_AIM' ? 'SUBSPACE' : 'MOUSE_AIM';
        
        // 画面上部に切り替え通知（シエロナビゲーションへの割り込みや画面一時テキストなど）
        Cielo.play(`フライト制御を${GAME.controlMode === 'MOUSE_AIM' ? '【マウスエイム】' : '【サブスペース】'}に変更します！`);
    }
});
```

### 3.3 自機の物理・操作の更新処理 (`update()` 内)
既存の `// --- Phase 2: 物理と操作 (Drift Physics) ---` および回転処理を以下のように修正・分岐する。

```javascript
            // --- Phase 2: 物理と操作 (Drift Physics) ---
            let ax = 0, ay = 0;
            // 既存のキー入力 ax, ay は、ADローテーションおよびWS推進に置き換わるため、
            // 物理演算は keys の入力状態を直接判定して、機首方向（bodyAngle）に基づいて推進を加算する。

            if (!GAME.isPlayerDying && !player.isLandingSequence && !GAME.launchSequence) {
                // A/Dキーでの機首旋回 (時計回り/反時計回り)
                if (keys['KeyA'] || keys['ArrowLeft']) {
                    player.bodyAngle -= 0.06; // 旋回スピード
                }
                if (keys['KeyD'] || keys['ArrowRight']) {
                    player.bodyAngle += 0.06;
                }

                // W/Sキーでの推進
                let thrust = 0;
                if (keys['KeyW'] || keys['ArrowUp']) thrust += playerStats.moveSpeed;
                if (keys['KeyS'] || keys['ArrowDown']) thrust -= playerStats.moveSpeed;

                if (thrust !== 0) {
                    player.vx += Math.cos(player.bodyAngle) * thrust;
                    player.vy += Math.sin(player.bodyAngle) * thrust;
                }
            }

            // --- ブースター (SHIFT) ---
            // 現在の機首方向 (bodyAngle) を基準にするため、既存 of ブースター計算コードはそのまま動作する。
```

### 3.4 砲塔および機首とマウスの連動処理 (`update()` 内)
```javascript
            // 機首と砲塔の回転制御 (着艦・発進シークエンス以外)
            if (!player.isLandingSequence && !GAME.launchSequence) {
                const dxMouse = mouse.x - GAME.width / 2;
                const dyMouse = mouse.y - GAME.height / 2;
                const angleToMouse = Math.atan2(dyMouse, dxMouse);

                if (GAME.controlMode === 'SUBSPACE') {
                    // サブスペースモード: 機首がマウス方向を追いかけ、砲塔は正面固定
                    player.bodyAngle = angleToMouse;
                    player.turretAngle = player.bodyAngle;
                } else {
                    // マウスエイムモード: 砲塔がマウス方向を向き、機首はキーボード(A/D)でのみ回転
                    player.turretAngle = angleToMouse;
                }
            }
```

### 3.5 射撃処理での参照角度
- 自機のバルカンや誘導ミサイルの射出時に使用する発射角について、常に `player.turretAngle` を基準にする（サブスペースモード時も自動的に `bodyAngle` と一致するため、既存の弾丸射出ロジックを変更することなく完全に動作する）。

## 4. 期待される効果
- **マウスエイムモード**: 「移動方向のコントロール（A/Dで旋回、W/Sで前後）」と「狙う方向のコントロール（マウス）」が完全に独立。キーを押し続けながら特定の角度に美しく滑りこみ、側面からバルカンを浴びせる「旋回スライド射撃」が可能。
- **サブスペースモード**: マウスで敵を視認して狙うだけで機体が自動的にその方向を向き、Wキーで突撃、Sキーで後退しながら「正面の敵を捉え続ける強烈な引き撃ちドリフト」を実現。
- **不具合極小化の保証**: 両モードとも推進ロジックが `bodyAngle` 依存に統合されるため、挙動の破綻がなく、摩擦モデルが単一のままコントロールの味付けだけを綺麗に変更可能。
