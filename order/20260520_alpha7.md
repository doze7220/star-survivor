# エンジントレイル（リボントレイル）描画エンジン刷新 仕様定義書 (VECTOR PATH STREAM PROTOCOL)

## 1. 概要
本ドキュメントは、自機および敵機のスラスター軌跡（リボントレイル）の描画方式を、従来の「独立した短線の連続描画方式」から、Canvasのパス機能（`lineTo`）を用いた「一繋ぎのポリゴンポリラインパス方式」へと刷新し、ベクターグラフィックのような滑らかで高輝度なライン表現を実現するための変更仕様を定義する。

---

## 2. 現状の課題分析
1. **節節目の目立ち（カクつき）**: 
   各履歴点をループで回し、毎回 `beginPath()` から `stroke()` までを独立して実行しているため、速度上昇時に点のサンプリング間隔が広がり、ラインの太さの変化（グラデーション）部分で不自然な段差やブツブツ感が発生している。
2. **描画負荷の冗長性**: 
   1つのトレイルに対して何度も描画コール（`stroke`）が発生しており、描画パフォーマンス低下の原因になり得る。

---

## 3. 変更仕様・定義

### 3.1 描画アルゴリズムの刷新（単一パスストリーム方式）
トレイルの全履歴を一繋ぎの1本のパス（Path）として構築し、**一回の描画コール（`stroke()`）で滑らかにレンダリング**する。

 **ベクター化のためのアプローチ**:
  1. トレイル履歴の最初の点へ `moveTo` で移動する。
  2. 2点目以降を順に `lineTo` で接続し、1本の滑らかなストリームラインを形成する。
  3. 太さ（`lineWidth`）はパス全体で一様になるため、 Canvas の `globalAlpha` や `strokeStyle` のグラデーションを活かして先端へ向かうフェードアウト（消えゆく光条）を美しく表現する。

---

## 4. 実装コード（差分・置き換え定義）

### 【修正箇所】draw() 関数内 `drawRibbonTrail` の全面書き換え
`draw()` 関数内（686行目付近〜726行目付近）にある既存の `drawRibbonTrail` 関数を、以下のベクターパス最適化コードに丸ごと置き換える。

```javascript
            // --- [修正] ベクターパスストリーム方式による滑らかなトレイル描画 ---
            function drawRibbonTrail(history, colorBase, maxLen) {
                if (!history || history.length < 2) return;

                ctx.save();
                
                // トレイルの先端（直近の座標）の状態を基準にする
                const latestPt = history[history.length - 1];
                // ブースト時は太さを2.5倍にし、SF的なエネルギー出力を表現
                const finalLineWidth = latestPt.boost ? maxLen * 2.5 : maxLen;

                // ─── 1. 外周の発光メインライン（単一パス接続） ───
                ctx.beginPath();
                ctx.lineCap = 'round';
                ctx.lineJoin = 'round'; // 結合部を滑らかに
                ctx.strokeStyle = colorBase;
                ctx.lineWidth = finalLineWidth;

                // 過去から現在に向かって一本のベクターパスを引く
                ctx.moveTo(history[0].x, history[0].y);
                for (let i = 1; i < history.length; i++) {
                    ctx.lineTo(history[i].x, history[i].y);
                }
                
                // 全体の透明度設定（スラスターのアクティブ状態に応じてブレンド）
                ctx.globalAlpha = latestPt.active ? 0.7 : 0.25;
                ctx.stroke();

                // ─── 2. 内側の高輝度ホワイトコアライン（レーザーのようなSF感の演出） ───
                ctx.beginPath();
                ctx.strokeStyle = '#ffffff';
                ctx.lineWidth = finalLineWidth * 0.35; // 芯の部分だけ白く発光

                ctx.moveTo(history[0].x, history[0].y);
                for (let i = 1; i < history.length; i++) {
                    ctx.lineTo(history[i].x, history[i].y);
                }
                
                ctx.globalAlpha = latestPt.active ? 0.9 : 0.3;
                ctx.stroke();

                ctx.restore();
            }
            // --- [修正終了] ---

5. 期待される効果
完全なベクター美:
自機がどれだけ超高速で移動・急旋回しても、線が途切れたりブツブツになったりせず、まるでイラストレーターで描いたような一本の美しい光条線が宇宙空間に描かれる。

描画負荷の大幅な軽減:
ループ内の stroke() コールが排除され、1つのトレイルにつき2回のコンテキスト描画で完結するため、敵の数が最大（同時5機など）になっても描画が一切重くならず、極めてクリーンに動作する。

```javascript
                // --- [修正] 敵機用 サブスペース慣性ドリフト推進ロジック ---
                const moveAngle = Math.atan2(targetY - e.y, targetX - e.x);

                let maxSpd = CONFIG.ENEMY_MAX_SPEED;
                if (e.personality === 'RAMMER') maxSpd *= 1.25;
                if (e.personality === 'SNIPER') maxSpd *= 0.85;

                // 敵の向いているアングルを移動方向（moveAngle）に合わせる
                // 障害物回避（avoidX, avoidY）のステアリング力を加味した理想の最高速度ベクトルを計算
                const targetVx = Math.cos(moveAngle) * maxSpd + avoidX;
                const targetVy = Math.sin(moveAngle) * maxSpd + avoidY;

                // 自機と同じブレンド係数（0.04）を適用し、既存の慣性を残しながらジワジワと目標へ引っ張る
                e.vx += (targetVx - e.vx) * 0.04;
                e.vy += (targetVy - e.vy) * 0.04;

                // Friction (宇宙空間の自然減衰)
                e.vx *= CONFIG.FRICTION;
                e.vy *= CONFIG.FRICTION;

                const enemySpeed = Math.hypot(e.vx, e.vy);
                if (enemySpeed > maxSpd) {
                    e.vx = (e.vx / enemySpeed) * maxSpd;
                    e.vy = (e.vy / enemySpeed) * maxSpd;
                }

                // 座標更新
                e.x += e.vx;
                e.y += e.vy;
                // --- [修正終了] ---


# 速度感応型スリップストリーム（風トレイル）システム 実装仕様定義書

## 1. 概要
本ドキュメントは、自機が一定速度以上で慣性移動（ドリフト含む）している際、空間の抵抗や「風」を視覚化する半透明の白いエフェクト（スリップストリームトレイル）を動的に生成・描画するシステム仕様を定義する。
このトレイルは、機体の向き（bodyAngle）ではなく、「実際の移動ベクトルの逆方向」へと流れることで、ドリフト時のスリップアングル（横滑り）を視覚的に強調する。

---

## 2. アルゴリズム・物理仕様
1. **発動条件（速度閾値）**:
   自機の現在の移動速度（`Math.hypot(player.vx, player.vy)`）が、通常最高速度の50%（約 `6.0` 以上）の時のみ風トレイルを生成する。
2. **描画ベクトル（風の向き）**:
   トレイルが伸びる方向は、常に `-player.vx`、`-player.vy`（移動方向の真逆）とする。
3. **視覚的特徴**:
   主スラスターのシアンの光条とは異なり、こちらは「空間を切り裂く風」を表現するため、極めて細く、半透明度（`globalAlpha`）の高い白（`#ffffff`）のラインとして描画する。

---
