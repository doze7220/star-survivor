# 作業指示書：RENDERING_TITLE_RESULT_INTEGRATION_PATCH
## 役割
あなたは優秀なフロントエンドエンジニアです。  
すでに実行済みの「描画フェーズ分離指示書（簡略版）」に追加する形で、  
**タイトル画面とリザルト画面の描画処理を drawOverlay.js に統合**してください。

## 目的
main.js に残っているタイトル画面・リザルト画面の描画処理を、  
**画面全体を覆う UI（Overlay）として drawOverlay.js にまとめる**。

**ゲームロジック・物理挙動・敵 AI・プレイヤー操作には一切影響を与えないこと（完全互換）。**  
**update() の内容には絶対に触れないこと。**

本作業は、既存の描画分離構造を拡張するための追加パッチである。
**記録**: 作業終了後、作業内容は `changelog.js` に記載し、リビジョン番号を上げること。

---

# 1. 対象ファイル
- `main.js`
- `js/utils/drawOverlay.js`

---

# 2. 移動対象（タイトル＋リザルト描画）

## 2-1. タイトル画面の描画
main.js 内に残っている **タイトル画面の描画処理** を抽出し、  
drawOverlay.js に移動する。

### 含める処理
- タイトルロゴ  
- タイトル演出（背景暗転・テキスト表示など）  
- 「PRESS ANY KEY」等の点滅テキスト  
- タイトル専用の UI 表示  

### 注意
- タイトル画面の **update ロジック（演出タイマーなど）** は main.js に残す。  
- **描画のみ** を移動する。

---

## 2-2. リザルト画面の描画
main.js 内に残っている **リザルト画面の描画処理** を抽出し、  
drawOverlay.js に移動する。

### 含める処理
- リザルトの背景  
- 明細行の描画  
- スタンプ演出  
- シエロのテキスト描画  
- 選択肢（RETURN TO GARAGE / RETRY MISSION）  
- 点滅カーソル  

### 注意
- リザルト画面の **update ロジック（行の進行・タイマー・選択状態）** は main.js に残す。  
- **描画のみ** を移動する。

---

# 3. drawOverlay.js の拡張

## 3-1. TITLE / RESULT の描画分岐を追加
drawOverlay.js の先頭に以下の構造を追加する：

if (GAME.state === 'TITLE') {
// タイトル画面の描画処理
return;
}

if (GAME.state === 'RESULT') {
// リザルト画面の描画処理
return;
}


※ コード例は書かないこと。

---

## 3-2. 既存のフェード・カウントダウン描画はそのまま維持
TITLE / RESULT の描画は **フェード・カウントダウンより優先**して return する。

---

# 4. main.js の変更

## 4-1. タイトル描画の削除
main.js 内に残っているタイトル描画コードを削除する。

## 4-2. リザルト描画の削除
main.js 内に残っているリザルト描画コードを削除する。

## 4-3. draw() の変更は不要
すでに drawOverlay() を呼び出す構造になっているため、  
draw() の変更は不要。

---

# 5. 完了条件
- タイトル画面の描画が drawOverlay.js に統合されていること。
- リザルト画面の描画が drawOverlay.js に統合されていること。
- main.js からタイトル描画・リザルト描画が完全に削除されていること。
- ゲーム実行時、タイトル・リザルト画面の見た目・挙動がリファクタリング前と完全一致すること。
- update() や物理挙動に一切の変更がないこと。
- 既存の描画分離構造（drawBackground / drawEffects / drawHUD / drawOverlay）が維持されていること。

---

# 6. 注意事項
- **コード例は一切生成しないこと。**  
- **update() の中身には絶対に触れないこと。**
- **プレイヤー描画・敵描画は移動禁止。**
- **タイトル・リザルトのロジック（状態遷移・タイマー）は移動禁止。**
