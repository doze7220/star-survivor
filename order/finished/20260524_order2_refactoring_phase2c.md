# タイトル・リザルト画面のシーン分割リファクタリング指示書

## 【役割】
あなたは優秀なフロントエンドエンジニアです。
main.js にある「タイトル画面」および「リザルト画面」の描画・更新ロジックを抽出し、独立したシーンとして切り出してください。

## 【絶対原則】
* ゲームの操作感や見た目、表示内容は一切変更しないこと（完全互換の維持）。
* シーン間の遷移ロジックは main.js（指揮者）に保持し、各シーンは自身の「描画」と「状態更新」のみに責任を持つこと。

## 【作業内容】
1. **ディレクトリ作成**: `js/scenes/` フォルダを作成すること。
2. **TitleScene.js の作成**:
    * `main.js` からタイトル画面に関連する全てのロジック（更新・描画・入力処理）を抽出し、`TitleScene` クラス（またはオブジェクト）として `js/scenes/TitleScene.js` に実装すること。
    * `update()` と `draw(ctx)` メソッドを持つこと。
3. **ResultScene.js の作成**:
    * `main.js` からリザルト画面に関連する全てのロジックを抽出し、`js/scenes/ResultScene.js` に実装すること。
    * `update()` と `draw(ctx)` メソッドを持つこと。
4. **main.js の改修**:
    * `main.js` 内の該当コードを削除し、タイトルやリザルトの状態に応じて `TitleScene` または `ResultScene` をインスタンス化し、メインループから `currentScene.update()` および `currentScene.draw(ctx)` を呼び出すように変更すること。
5. **HTML設定**: `index.html` に新しいシーン用スクリプトの読み込みを追加すること。

## 【設計指針】
* 各シーンは `class` または `Object` として定義し、他のシーンや main.js から独立して動作するように実装すること。
* データやゲームの状態が必要な場合は、引数で渡すか、グローバルな GAME オブジェクトから取得すること。

## 【補足】
* 必要な定数やユーティリティ関数が不足している場合は、utils.js を適切に参照するように修正すること。
* 作業内容は `meta/changelog.js` に記載し、リビジョン番号を上げること。
