# 『STAR SURVIVOR』コントロールシステム拡張指示書 (Alpha 3)

本ドキュメントは、自機のフライトモデル（操作性および慣性挙動）に、伝統的な宇宙慣性ドリフトシューター『SubSpace』の操作性を追加し、「マウスエイムモード」と「サブスペースモード」をキーボード操作でリアルタイムに切り替え可能にするための実装指示書である。

ユーザーフィードバックに基づき、**「旋回速度のスラスターハンドリング（HANDLING）ステータス連動」**、および**「サブスペースモード時のマウス追従とキーボード入力（A/D）の競合解決仕様」**を明確化して策定。

---

## 1. コア仕様

### 1.1 共通物理モデル
- **摩擦係数 (`CONFIG.FRICTION`)**: 双方のモードで共通の値（現在の `0.998`）を使用する。モード切り替え時に摩擦自体の係数は変更しない。

### 1.2 プレイヤーステータス「スラスターハンドリング（HANDLING）」の連動
旋回速度を一定値（ハードコード）にするのではなく、未実装だった `HANDLING` ステータスと連動させ、レベルアップやアップグレードで強化可能にする。
- **設定値の追加 (`CONFIG`)**:
  `PLAYER_BASE_HANDLING: 0.06,  // 自機の初期旋回速度 (ラジアン/フレーム)`
- **アップグレード定義の拡張 (`playerStats.upgrades`)**:
  `playerStats.upgrades.handling = 1;`（初期レベル）
- **動的ステータス算出 (`playerStats.handling`)**:
  毎フレームのアップデート開始時に以下を計算する：
  `playerStats.handling = CONFIG.PLAYER_BASE_HANDLING + (playerStats.upgrades.handling - 1) * 0.015;`
- **UI表示の連動**:
  画面右下のステータスパネル `// THRUSTER` の `HANDLING` 項目へ値を反映する：
  `Lv${playerStats.upgrades.handling} ${(playerStats.handling * 60).toFixed(1)}°/s` (60fps換算の角度表示など)

### 1.3 コントロールモードの定義と切り替え
- ゲームのステートに `GAME.controlMode = 'MOUSE_AIM';`（デフォルト）を追加。
- プレイ中に **`C` キー（KeyC）** を押すことで、以下の2つのモードが交互に瞬時に切り替わる。
  - `'MOUSE_AIM'`（マウスエイムモード）
  - `'SUBSPACE'`（サブスペースモード）
- 切り替え成功時、Cieloが現在のモードをボイス（通信テキスト）で通知する。

## 2. 各モードの操作仕様

### 2.1 マウスエイムモード (MOUSE_AIM)
従来の360度自由エイム射撃を活かしつつ、旋回操作をキーボードにアサインしたモード。
- **旋回操作 (A / D キー)**:
  - **Aキー（左）**: 自機の機首（`player.bodyAngle`）を反時計回りに `playerStats.handling` の速度で旋回させる。
  - **Dキー（右）**: 自機の機首（`player.bodyAngle`）を時計回りに `playerStats.handling` の速度で旋回させる。
- **推力操作 (W / S キー)**:
  - **Wキー（前進）**: 現在の機首の向き（`player.bodyAngle`）に向かって推進加速度を加算。
  - **Sキー（後退・ブレーキ）**: 現在の機首の向きの真後ろに向かって推進加速度を加算。
- **砲塔制御 (マウス移動)**:
  - マウスカーソルのある方向へ、砲塔（`player.turretAngle`）が360度自在に旋回・追従。
- **射撃**:
  - 砲塔の向いている方向（`player.turretAngle`）へバルカンおよびミサイルを発射。

### 2.2 サブスペースモード (SUBSPACE)
名作『SubSpace』の挙動を再現し、機首方向へエイムと推進が完全同期する、慣性ドリフトに特化した本格宇宙機動モード。
- **旋回操作 (A / D キー)**:
  - **A/Dキー** で機体（`player.bodyAngle`）を `playerStats.handling` の速度で旋回させる。
- **推力操作 (W / S キー)**:
  - **W/Sキー** で現在の機首方向の前後に向かって推進する（※マウスエイムモードと共通）。
- **砲塔制御 (自機正面固定)**:
  - 砲塔は完全に機首正面（`player.bodyAngle`）にロック・固定される。
  - `player.turretAngle = player.bodyAngle;`
- **射撃**:
  - 機首の正面方向（`player.bodyAngle`）へ真っ直ぐバルカンおよびミサイルを発射。

## 🚀 3. 【重要】サブスペース操作時の「マウス追従 ⇄ キー入力」競合解決仕様

サブスペース操作時にマウスによる機首追従と、キーボード（A/Dキー）での手動旋回が同時に入力された場合の処理については、ゲームプレイの快適さと操作性を最大化するため、以下の **【解決策A (推奨)】** を基本仕様とする。

### 【解決策A】キーボード優先・マウス一時的オーバーライド（Keyboard Priority with Mouse Lock）
- **動作ルール**:
  1. **プレイヤーが A または D キーを「押している」間**:
     - マウスによる機首回転（自動追従）は**完全に一時停止（無視）**されます。
     - 自機の旋回はキーボード入力（Aキーで反時計回り、Dキーで時計回り）のみによって、`playerStats.handling` の角速度で制御されます。
  2. **プレイヤーが A および D キーを「離した」瞬間**:
     - 即座にマウス自動追従が再開され、機首はスムーズにマウスカーソルがある角度へ向かって回転します。
- **採用理由**:
  - 狙いたい獲物が画面内にいる時、マウスエイムによる自動追従で直感的な射撃ができる一方、「大きく素早く旋回したい時」や「慣性を維持してアステロイドのようにグルグル回転しながらドリフトしたい時」には、キーボードを押し込むだけでマウスの微細なブレに邪魔されず、100%意図通りの手動旋回に移行できるため、操作の自由度が最も高くなります。

### 【予備の解決策B】対抗スラスター（角速度の競合合算）モデル
- もし物理的な相殺を好む場合、マウスが指示する回転方向（時計回り `+1` / 反時計回り `-1`）と、キー入力（Dで `+1` / Aで `-1`）を合算し、その結果の方向に `playerStats.handling` で回す。同方向なら最大速度、逆方向なら互いに打ち消し合って静止する。
- ※実装がやや複雑になり操作の「引っかかり」に繋がる可能性があるため、まずは極めて直感的かつ確実な **【解決策A】** で実装する。

## 4. 実装ガイドライン（コード変更ポイント）

### 4.1 `playerStats` の拡張 (`index.html`)
```javascript
        // 初期定義部分 (playerStats)
        const playerStats = {
            // ...既存項目
            upgrades: {
                fireRate: 1,
                bulletCount: 1,
                moveSpeed: 1,
                handling: 1 // ★追加
            }
        };
```

### 4.2 ステータス動的計算 (`update()` 開始部分)
```javascript
        function update() {
            // ...既存のレベルスケール処理など
            
            // ★ハンドリング値の計算 (CONFIG.PLAYER_BASE_HANDLING は 0.06 に定義)
            playerStats.handling = CONFIG.PLAYER_BASE_HANDLING + (playerStats.upgrades.handling - 1) * 0.015;
```

### 4.3 物理と操作の更新処理 (`update()` 内)
```javascript
            // --- Phase 2: 物理と操作 (Drift Physics) ---
            if (!GAME.isPlayerDying && !player.isLandingSequence && !GAME.launchSequence) {
                // A/Dキーでの機首旋回 (共通)
                if (keys['KeyA'] || keys['ArrowLeft']) {
                    player.bodyAngle -= playerStats.handling; // ★ハンドリング値を参照
                }
                if (keys['KeyD'] || keys['ArrowRight']) {
                    player.bodyAngle += playerStats.handling; // ★ハンドリング値を参照
                }

                // W/Sキーでの推進 (共通)
                let thrust = 0;
                if (keys['KeyW'] || keys['ArrowUp']) thrust += playerStats.moveSpeed;
                if (keys['KeyS'] || keys['ArrowDown']) thrust -= playerStats.moveSpeed;

                if (thrust !== 0) {
                    player.vx += Math.cos(player.bodyAngle) * thrust;
                    player.vy += Math.sin(player.bodyAngle) * thrust;
                }
            }
```

### 4.4 機首と砲塔の回転制御 (`update()` 内)
【解決策A】に基づき、A/Dキー押下中はマウスによる追従をロック（無視）するロジックを実装。

```javascript
            // 機首と砲塔の回転制御 (着艦・発進シークエンス以外)
            if (!player.isLandingSequence && !GAME.launchSequence) {
                const dxMouse = mouse.x - GAME.width / 2;
                const dyMouse = mouse.y - GAME.height / 2;
                const angleToMouse = Math.atan2(dyMouse, dxMouse);

                if (GAME.controlMode === 'SUBSPACE') {
                    // サブスペースモード:
                    // 1. A/Dキーによる手動回転キーが「押されていない」時のみ、マウス方向へスムーズ自動旋回
                    const isRotatingKeysPressed = keys['KeyA'] || keys['ArrowLeft'] || keys['KeyD'] || keys['ArrowRight'];
                    
                    if (!isRotatingKeysPressed) {
                        // マウス角度へ向けて smooth 旋回
                        let diff = angleToMouse - player.bodyAngle;
                        // 角度を [-PI, PI] の範囲にクランプ
                        diff = Math.atan2(Math.sin(diff), Math.cos(diff));
                        
                        if (Math.abs(diff) < playerStats.handling) {
                            player.bodyAngle = angleToMouse;
                        } else {
                            player.bodyAngle += Math.sign(diff) * playerStats.handling;
                        }
                    }
                    
                    // 砲塔は常に正面固定
                    player.turretAngle = player.bodyAngle;
                } else {
                    // マウスエイムモード: 砲塔がマウス方向を向き、機首はキーボード(A/D)でのみ回転
                    player.turretAngle = angleToMouse;
                }
            }
```

## 5. UIでのステータス反映の変更
画面右下の `// THRUSTER` パネル内の `HANDLING` 行を以下のように書き換えて、ハンドリング性能の視覚的強化を実感できるようにする：
```javascript
`<div class="stat-row"><span class="stat-label">HANDLING</span><span class="stat-value">Lv${playerStats.upgrades.handling} ${(playerStats.handling * 60).toFixed(1)}°/s</span></div>`
```

これにより、プレイヤーはフライト感（滑らかな旋回）が自分のアップグレードでダイナミックに鋭くなっていく楽しさを実感できるようになる。
