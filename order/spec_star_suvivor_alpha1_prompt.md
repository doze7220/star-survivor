STAR SURVIVER：AI開発用アルファ版（MVP）実装プロンプト v.2
1. プロジェクト概要と設計思想
本ドキュメントは、レトロスタイルの宇宙空間シューティングゲーム『STAR SURVIVER』のアルファ版（MVP）を開発するための厳密な技術指示書である。エンジニアリングの観点から、AIが生成すべきコードの品質と構造を定義する。

アルファ版の評価基準と優先順位は `documents/spec_star_suvivor_alpha_focus.md` を正とする。ここでは、その評価基準を満たすための実装詳細を扱う。
コンセプトの要約
タイトル： STAR SURVIVER（スター・サバイバー）
ジャンル： 見下ろし型宇宙空間シューティング
スタイル： レトロスタイル（『SubSpace』『ボスコニアン』『タイムパトロール』の操作感と視覚効果を完全継承）
ゲーム性： 『Vampire Survivors』ライクな成長システム（敵撃破、経験値取得、レベルアップ時の三者択一強化）
技術スタックおよび制約
環境： HTML5, JavaScript (Canvas API)
依存関係： 外部ライブラリ（Phaser, PixiJS等）の使用は一切禁止。バニラなJavaScriptのみで実装すること。
構成： 開発効率とポータビリティを重視し、HTML, CSS, JavaScriptを単一のHTMLファイル内に完結させること。

2. 描画システムの最適化：SpriteCacheの導入
毎フレームの描画負荷を最小化するため、以下のアーキテクチャを強制する。
SpriteCache オブジェクトの実装
メインループ内での arc() や lineTo() の直接呼び出しを禁止し、初期化時にオフスクリーンキャンバスへ描画・蓄積する SpriteCache 構造を採用せよ。
実装要件：
AssetManager または SpriteCache オブジェクトを生成し、自機、敵機、ショット、パーティクルを事前に描画する。
描画時、機体の回転中心を整合させるため、必ずキャッシュサイズの半分をオフセットした ctx.drawImage(cache, -width / 2, -height / 2) を実行すること。これにより回転軸のズレを完全に排除する。
--------------------------------------------------------------------------------
3. エンティティのビジュアルと方向制御
機体デザインと回転ロジック
ビジュアル： 自機および敵機は「▲（二等辺三角形）」を基本とし、進行方向の頂点付近に黒い円形「●（キャノピー）」を描画せよ。
回転処理： ctx.save() -> ctx.translate(x, y) -> ctx.rotate(angle) -> drawImage -> ctx.restore() のフローを厳守すること。
角度計算：
自機：マウス座標または移動ベクトルから Math.atan2(dy, dx) を用いて算出。
敵機：自機の座標を追跡し、常にターゲットの方向へ回転角を更新する。
--------------------------------------------------------------------------------
4. 環境演出：多重スクロールとパーティクル
3層パララックス・スターフィールド
奥行きを表現するため、背景の星々（Starfield）を以下の係数に基づき管理せよ。
Layer 1 (遠景): scrollRate: 0.2 / サイズ小 / 低速
Layer 2 (中景): scrollRate: 0.5 / サイズ中 / 中速
Layer 3 (近景): scrollRate: 0.8 / サイズ大 / 高速
ロジック： 星の座標更新は star.x -= player.vx * layer.scrollRate のように、プレイヤーの移動速度に係数を乗じて計算すること。
慣性パーティクルシステム
スラスターの演出として、以下の挙動を実装せよ。
慣性の継承： 生成時、機体の現在の移動ベクトルを一定割合継承すること。
減衰： 毎フレーム alpha（不透明度）を減少させ、同時にサイズを縮小（サイズダウン）させることで、自然な消滅を表現せよ。
--------------------------------------------------------------------------------
5. ゲームプレイ・メカニクスと成長システム
移動物理（Drift Physics）
「滑り」のある操作感を実現するため、以下の物理パラメータを適用せよ。
摩擦係数（Friction）： 0.98 を毎フレーム速度に乗算。
加速度： キー入力により速度ベクトルに加算。
最大速度： Math.hypot(vx, vy) を用いて閾値を監視し、クランプ処理を行うこと。
成長システムのデータ構造
『Vampire Survivors』のアップグレード要素を再現するため、以下のデータ構造を基盤として実装せよ。
const playerStats = {
    level: 1,
    exp: 0,
    nextLevelExp: 100,
    fireRate: 10,   // 連射速度（フレーム間隔）
    bulletCount: 1, // 同時発射数
    moveSpeed: 2    // 加速度係数
};

const upgradePool = [
    { id: 'fireRate', name: 'Rapid Fire', description: 'Increase fire rate by 20%' },
    { id: 'bulletCount', name: 'Multi-Shot', description: 'Add extra bullet' },
    { id: 'moveSpeed', name: 'Engine Boost', description: 'Increase movement speed' }
];
レベルアップ・シーケンス： exp >= nextLevelExp に達した際、ゲームを一時停止（Pause）し、upgradePool からランダムに選ばれた3つの選択肢を画面に提示する UI ステートを実装せよ。

6. 実装フェーズとディレクトリ構造
以下の順序で実装を進めること。
Phase 1: 描画基盤： キャンバス初期化、SpriteCache によるアセット事前生成、メインループ構築。
Phase 2: 物理と操作： 慣性移動ロジック（Friction 0.98）と、キャノピー付き機体の回転描画。
Phase 3: 背景と演出： 3層パララックス（0.2, 0.5, 0.8）とスラスターパーティクルの実装。
Phase 4: 衝突判定と敵AI： Math.hypot を用いた円形衝突判定の実装。敵機の単純追尾アルゴリズムの実装。
Phase 5: 成長要素： 経験値ドロップの回収、レベルアップ時の「一時停止・三者択一」強化システムの構築。
--------------------------------------------------------------------------------
7. 制約および禁止事項

外部リソース禁止： 全てのグラフィックは Canvas API によるプログラム描画のみで完結させること。画像ファイルの読み込み（<img>タグやURL参照）は一切禁止する。
