# 作業指示書：20260524_order3_refactoring_phase2g_COMMUNICATION
## 役割
あなたは優秀なフロントエンドエンジニアです。  
提供したソースコードに対し、以下の「通信システム（Cielo）の軽量リファクタリング作業」を行ってください。

## 目的
main.js に直書きされている Cielo 通信 UI の処理を、  
専用クラスへ完全に分離し、UI レイヤーとして適切にカプセル化する。

**ゲームロジック・物理挙動・敵 AI・プレイヤー操作には一切影響を与えないこと（完全互換）。**  
**update() の内容には絶対に触れないこと。**

本作業は Newtonian Symmetry（物理統合）フェーズの前処理である。
**記録**: 作業終了後、作業内容は `js/changelog.js` に記載し、リビジョン番号を上げること。

---

# 1. 新規ファイルの作成
以下のファイルを作成する：

- `js/classes/ui/communication.js`

このファイルには、**通信 UI を管理するクラス**を定義する。  
クラス名は以下のいずれかとする：

- `Communication`
- `CommSystem`
- `CommUI`

（Cielo 固有ではなく、将来の複数キャラ通信を想定した抽象名とする）

---

# 2. クラスの仕様
以下の責務を持つクラスを作成する。

## 2-1. コンストラクタ
- DOM 要素 `#cielo-comm` を取得して内部に保持する  
  ※ 現時点では Cielo 固定だが、将来の拡張を妨げない構造にする
- タイマー管理用のプロパティを初期化する
- 現在表示中のメッセージを保持するプロパティを初期化する

## 2-2. メッセージ表示メソッド
### `play(msg, type = "normal")`
- 既存の Cielo.play と **完全に同じ挙動** を再現する  
  （表示時間、色変更、フェードアウト、重複メッセージ抑制など）
- 表示時間は `msg.length * 1000` をそのまま使用する  
  ※ 挙動互換のため変更禁止
- 色変更ロジックは type に応じて行う  
  - `"normal"` → 緑  
  - `"system"` → 赤  
- タイマーのクリアと再設定を適切に行う

## 2-3. 内部メソッド
以下のような内部処理を private 的に分離してよい：

- 色変更処理
- メッセージ DOM 更新処理
- フェードアウトタイマー処理

※ 名前や構造は自由。  
※ 外部 API として公開するのは `play()` のみ。

---

# 3. main.js の変更
以下の変更を行う。

## 3-1. 旧 Cielo オブジェクトの削除
main.js に存在する Cielo オブジェクト定義を **完全に削除**する。

## 3-2. 新クラスのインポートとインスタンス化
main.js の先頭付近に以下を追加する：

- `import Communication from "./js/classes/ui/communication.js";`
- `const comm = new Communication();`

## 3-3. 既存の Cielo.play 呼び出しの置換
main.js 内のすべての

Cielo.play(...)

を

comm.play(...)


に置換する。

### ※重要
- **引数・メッセージ内容・呼び出しタイミングは絶対に変更しないこと。**
- これにより完全互換が維持される。
